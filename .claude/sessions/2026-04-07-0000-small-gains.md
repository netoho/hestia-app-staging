### Lessons Learned
- For auto-incrementing IDs, prefer Postgres sequences over app-level `MAX+1` â€” avoids concurrency bugs and matches `@default(dbgenerated(...))` cleanly.
- Delegate validation logic (e.g., actor completeness) to a single service; avoid UI-level reimplementation.
- Status codes should flow from `Result.error.statusCode` through API routes, not be hardcoded.
